## 2. 之前的记录方式（共性）

- 以 Markdown 长文为主，按“流程/模块/接口”分段。
- 常见结构是：
  - 目标或背景
  - 路由或 API 列表
  - 请求/响应示例
  - 约束规则或状态机
- 存在“规范/计划文档”和“当前实现”混写的情况，时间久了会偏离代码。
- 绝大多数文档没有明确写“代码真值来源文件”与“更新时间戳”。

## 3. 本次补充文档策略

- 新增一份“当前实现快照”文档，明确：
  - 生效逻辑来自哪些代码文件；
  - 关键参数（并发、重试、超时、页数约束）当前值是多少；
  - 已知偏差/待修项有哪些。
- 保留历史文档，不覆盖旧方案文档，避免丢失演进过程。
